# SecAlert 项目落地方案

## 一、项目概述

### 1.1 项目背景

大型企业网络中部署了大量异构安全设备，日志告警格式各异、数量巨大，难以关联分析进行威胁检测。
本项目旨在构建一套告警分析系统，解决数据孤岛问题，还原攻击链，提升安全运营效率。

### 1.2 核心约束

| 约束项   | 内容                      |
| ----- | ----------------------- |
| 客户类型  | 大型企业                    |
| 数据规模  | 10万+ 告警/天               |
| 部署环境  | 完全私有化离线部署               |
| 质量要求  | 精确性极高（误报会失去用户信任）        |
| AI 推理 | Vllm + Qwen3-32B（已离线部署） |

### 1.3 第一期目标

- 全部基本功能打通，具备完整的告警分析能力
- 第二期进行高级功能扩展（API/MCP服务、AI Agent）

class LogParserSignature(dspy.Signature):

#### 4.4.2 DSPy 程序

```python
class LogParserProgram(dspy.Module):
    def __init__(self):
        super().__init__()
        self.generate = dspy.Treact(LogParserSignature)

    def forward(self, raw_logs: list[str], source_type: str = "未知"):
        result = self.generate(
            raw_logs=chr(10).join(raw_logs),
            source_type=source_type
        )
        return result
```

#### 4.4.3 优化器（MIPRO）

```python
class LogParserOptimizer:
    def compile(self, training_data: list):
        """
        基于高质量反馈数据编译优化
        - MIPRO: 贝叶斯优化搜索最优 prompt
        - num_trials: 50
        """
        optimizer = dspy.MIPRO(
            metric=self._eval_metric,
            num_trials=50,
            max_cost=1000
        )
        return optimizer.compile(self.model, trainset=training_data)

    def _eval_metric(self, example, prediction, trace=None):
        regex_match = self._check_regex(example, prediction)
        field_match = self._check_mappings(example, prediction)
        return (regex_match + field_match) / 2
```

#### 4.6.3 统一反馈优化机制

```python
class UnifiedFeedbackStore:
    """
    统一的反馈存储，同时服务解析任务和分析任务
    """

    def __init__(self):
        self.parser_feedback: list[ParserFeedback] = []
        self.analysis_feedback: list[AnalysisFeedback] = []

    # ---------- 解析任务反馈 ----------

    def add_parser_feedback(self, feedback: ParserFeedback):
        """记录解析结果反馈"""
        self.parser_feedback.append(feedback)
        self._auto_compile_if_needed()

    def get_parser_training_data(self, min_quality: float = 0.8) -> list:
        """获取高质量解析训练数据"""
        return [
            f for f in self.parser_feedback
            if f.is_correct and f.regex_score >= min_quality
        ]

    # ---------- 分析任务反馈 ----------

    def add_analysis_feedback(self, feedback: AnalysisFeedback):
        """记录分析结果反馈"""
        self.analysis_feedback.append(feedback)
        self._auto_compile_if_needed()

    def get_analysis_training_data(self, min_quality: float = 0.8) -> list:
        """获取高质量分析训练数据"""
        return [
            f for f in self.analysis_feedback
            if f.is_correct and f.confidence >= min_quality
        ]

    # ---------- 统一编译优化 ----------

    def compile_all(self, programs: dict[str, dspy.Module], vllm_client):
        """
        对所有 DSPy 程序进行编译优化
        """
        results = {}

        # 编译解析程序
        parser_data = self.get_parser_training_data()
        if len(parser_data) >= 10:
            parser_optimizer = LogParserOptimizer(vllm_client)
            results["parser"] = parser_optimizer.compile(parser_data)

        # 编译分析程序
        analysis_data = self.get_analysis_training_data()
        if len(analysis_data) >= 10:
            analyzer_optimizer = AlertAnalyzerOptimizer(vllm_client)
            results["analyzer"] = analyzer_optimizer.compile(analysis_data)

        return results

    def _auto_compile_if_needed(self):
        """根据反馈积累情况自动触发编译"""
        # 每积累 50 条新反馈 或 每 24 小时，触发一次编译
        pass
```

#### 4.6.4 反馈数据模型

```python
from pydantic import BaseModel
from datetime import datetime
from enum import Enum


class FeedbackSource(str, Enum):
    MANUAL_REVIEW = "manual_review"      # 人工审核
    AUTOMATED_TEST = "automated_test"   # 自动验证
    PRODUCTION = "production"           # 生产反馈


# ---------- 解析任务反馈 ----------
class ParserFeedback(BaseModel):
    id: str
    timestamp: datetime

    # 输入
    raw_log: str
    source_type: str

    # 期望输出
    expected_regex: str
    expected_mappings: dict

    # 实际输出
    actual_regex: str | None = None
    actual_mappings: dict | None = None

    # 评估
    is_correct: bool
    regex_score: float = 0.0  # 0-1
    mapping_score: float = 0.0  # 0-1

    # 元信息
    source: FeedbackSource = FeedbackSource.MANUAL_REVIEW
    reviewed_by: str | None = None


# ---------- 分析任务反馈 ----------
class AnalysisFeedback(BaseModel):
    id: str
    timestamp: datetime

    # 输入
    alert_event: dict
    related_events: list[dict]
    expected_result: str

    # 实际输出
    actual_result: str | None = None

    # 评估
    is_correct: bool
    confidence: float = 0.0  # 0-1

    # 元信息
    source: FeedbackSource = FeedbackSource.MANUAL_REVIEW
    analyst_id: str | None = None
    notes: str | None = None
```

## 六、分析层

### 6.1 告警分析层次

```
Level 1: 单点检测 ── 规则匹配、签名检测、阈值告警
    ↓
Level 2: 关联分析 ── 时序关联、跨源关联、统计异常
    ↓
Level 3: 高级分析 ── 图分析、DSPy AI、威胁狩猎
```

### 6.2 Level 1：规则检测（基础能力）

```yaml
rules:
  - name: "暴力破解检测"
    condition: |
      event.type == "auth_failure"
      AND count > 5 WITHIN 5min BY src_ip
    severity: high
    mitre_tactics: [TA0001]

  - name: "C2通信检测"
    condition: |
      event.domain IN威胁情报库.malicious_domains
    severity: critical
    mitre_techniques: [T1071]
```

### 6.3 Level 2：关联分析

| 类型 | 说明 |
|------|------|
| 时序关联 | 同一实体的连续异常行为 |
| 跨源关联 | 防火墙+IDS+EDR 多设备协同 |
| 统计异常 | 基线偏离、信息熵、PCA |

### 6.4 Level 3：DSPy AI 分析

DSPy 驱动的 AI 分析是高级分析的核心能力，与解析层共用同一套优化框架。

#### 6.4.1 告警上下文分析

```python
class AlertAnalyzerProgram(dspy.Module):
    def __init__(self):
        super().__init__()
        self.analyze = dspy.Treact(AlertAnalyzerSignature)

    def forward(self, alert: dict, related: list, intel: dict):
        return self.analyze(
            alert_event=json.dumps(alert),
            related_events=json.dumps(related),
            threat_intel=json.dumps(intel)
        )
```

#### 6.4.2 攻击链智能还原

```python
class AttackChainProgram(dspy.Module):
    def __init__(self):
        super().__init__()
        self.reconstruct = dspy.Treact(AttackChainSignature)

    def forward(self, events: list, graph: dict):
        return self.reconstruct(
            events=json.dumps(events, ensure_ascii=False),
            entity_graph=json.dumps(graph)
        )
```

#### 6.4.3 风险评分

```python
class RiskScoringProgram(dspy.Module):
    def __init__(self):
        super().__init__()
        self.score = dspy.Treact(RiskScoringSignature)

    def forward(self, entity: dict, history: list, context: dict):
        return self.score(
            entity=json.dumps(entity),
            behavior_history=json.dumps(history),
            alert_context=json.dumps(context)
        )
```

### 6.6 攻击链还原

DSPy AI 与规则引擎协同工作：

```
┌─────────────────────────────────────────────────────────────┐
│                    攻击链还原流程                            │
│                                                             │
│  告警事件 ──→ [规则引擎] ──→ 战术映射（ATT&CK）            │
│                   │                                         │
│                   ↓                                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │           [DSPy AttackChainProgram]                 │   │
│  │  输入：关联事件 + 实体关系图                          │   │
│  │  输出：攻击链 + 阶段判定 + 风险评分                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                   │                                         │
│                   ↓                                         │
│              攻击链可视化                                   │
└─────────────────────────────────────────────────────────────┘
```

#### 6.6.1 ATT&CK 驱动分析

```
检测告警 → 映射战术/技术 → 战术序列还原
                              ↓
        [初始访问] → [执行] → [持久化] → [横向移动] → [数据窃取]
                              ↓
                    时间线可视化 + 风险评分
```

#### 6.6.2 攻击链模板匹配

预定义攻击链模板与实际攻击路径匹配：

```
模板库：
- 勒索软件路径：[钓鱼] → [宏执行] → [横向移动] → [加密] → [勒索]
- APT持久化路径：[初始访问] → [提权] → [持久化] → [C2] → [凭据窃取]
- 内部威胁路径：[登录] → [敏感访问] → [批量下载] → [外发]

匹配：实际攻击链 → 与模板匹配度评分 → 输出攻击类型推断
```
