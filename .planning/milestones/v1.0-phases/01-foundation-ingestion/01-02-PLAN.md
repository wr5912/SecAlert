  <files>collector/configs/vector-kafka-sink.yaml</files>
  <read_first>
    - collector/configs/vector-kafka-sink.yaml (new file)
    - .planning/phases/01-foundation-ingestion/01-RESEARCH.md (Kafka sink config)
  </read_first>
  <action>
    Create collector/configs/vector-kafka-sink.yaml:

    ```yaml
    sinks:
      kafka_suricata:
        type: kafka
        inputs:
          - suricata_syslog
        bootstrap_servers: kafka:29092
        topic: raw-suricata
        compression: gzip                      # Reduce network/memory
        acks: all                              # Wait for all replicas (no data loss)
        batch_size: 16384                     # Tune for throughput
        linger_ms: 5                           # Small batch window for low latency
        retry_max_duration_secs: 300
        buffer:
          type: disk
          max_size: 1073741824                # 1GB disk buffer
          num_segments: 10
          when_full: block                    # Backpressure to prevent loss
    ```

    Per research: `acks: all` is critical for no data loss. `compression: gzip` reduces bandwidth. `batch_size: 16384` and `linger_ms: 5` balance throughput vs latency.
  </action>
  <verify>
    <automated>grep -E "acks:|compression:|batch_size:|linger_ms:" collector/configs/vector-kafka-sink.yaml</automated>
  </verify>
  <done>Vector Kafka sink configured with reliable delivery settings</done>
