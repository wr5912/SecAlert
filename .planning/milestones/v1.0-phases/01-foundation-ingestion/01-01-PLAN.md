<task type="auto">
  <name>Task 1: Create Docker Compose configuration</name>
  <files>docker-compose.yml</files>
  <read_first>
    - docker-compose.yml (new file - no existing content)
    - .planning/codebase/STACK.md (for service versions)
  </read_first>
  <action>
    Create docker-compose.yml with these services:

    1. **zookeeper** (image: confluentinc/cp-zookeeper:7.5.0)
       - port 2181
       - environment: ZOOKEEPER_CLIENT_PORT=2181, ZOOKEEPER_TICK_TIME=2000

    2. **kafka** (image: confluentinc/cp-kafka:7.5.0)
       - ports: 9092 (host), 29092 (internal)
       - depends_on: zookeeper
       - environment:
         - KAFKA_BROKER_ID=1
         - KAFKA_ZOOKEEPER_CONNECT=zookeeper:2181
         - KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://kafka:29092,PLAINTEXT_HOST://localhost:9092
         - KAFKA_LISTENER_SECURITY_PROTOCOL_MAP=PLAINTEXT:PLAINTEXT,PLAINTEXT_HOST:PLAINTEXT
         - KAFKA_INTER_BROKER_LISTENER_NAME=PLAINTEXT
         - KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR=1
         - KAFKA_AUTO_CREATE_TOPICS_ENABLE=true
       - healthcheck: kafka-topics --bootstrap-server localhost:9092 --list

    3. **vector** (image: timberio/vector:0.34.0)
       - ports: 514 (tcp), 515 (udp)
       - depends_on: kafka (service healthy)
       - volumes: ./collector/configs:/vector configs:ro
       - environment: VECTOR_CONFIG_DIR=/vector

    4. **elasticsearch** (image: docker.elastic.co/elasticsearch/elasticsearch:8.11.0)
       - ports: 9200
       - environment:
         - discovery.type=single-node
         - xpack.security.enabled=false
         - ES_JAVA_OPTS=-Xms512m -Xmx512m
       - healthcheck: curl -s http://localhost:9200/_cluster/health

    5. **redis** (image: redis:7-alpine)
       - port: 6379
       - command: redis-server --maxmemory-policy allkeys-lru --maxmemory 256mb
       - healthcheck: redis-cli ping

    6. **postgres** (image: postgres:16-alpine)
       - port: 5432
       - environment:
         - POSTGRES_DB=secalert
         - POSTGRES_USER=secalert
         - POSTGRES_PASSWORD=secalert_dev
       - volumes: ./storage/postgres/init.sql:/docker-entrypoint-initdb.d/init.sql
       - healthcheck: pg_isready -U secalert

    Use version "3.8" and create a "secalert-network" bridge network.

    Important: Kafka must start BEFORE vector (use service_healthy condition).
  </action>
  <verify>
    <automated>docker-compose config --services | grep -E "zookeeper|kafka|vector|elasticsearch|redis|postgres" | wc -l == 6 && echo "All 6 services defined"</automated>
  </verify>
  <done>docker-compose.yml defines all 6 services with proper dependencies, healthchecks, and network configuration</done>
