# Phase 19 Plan 01: 采集可观测性监控

## 执行目标

实现 SM-02 采集可观测性监控，包括 EPS/延迟/成功率监控、DLQ 大小监控和 Prometheus metrics 暴露。

## 需求回顾 (SM-02)

| 需求 | 说明 |
|------|------|
| EPS 监控 | 每秒事件数监控：in/out 双向计数 |
| 采集延迟监控 | collection_lag_ms 监控 |
| 解析成功率监控 | parse_success_rate 监控 |
| DLQ 大小监控 | dlq_size 监控 |
| 数据源健康监控 | datasource_health 监控 |
| Prometheus metrics | Prometheus metrics 端点暴露 |

## 任务分解

### Task 1: 创建 Metrics 数据模型
**文件:** `src/collection/metrics/models.py`

**指标定义:**
```python
@dataclass
class CollectionMetrics:
    # 计数指标 (Counter)
    events_in_total: int = 0           # 总输入事件数
    events_out_total: int = 0         # 总输出事件数
    parse_errors_total: int = 0       # 解析错误总数
    dlq_messages_total: int = 0       # DLQ 消息总数

    # 速率指标 (计算得出)
    events_in_rate: float = 0.0       # EPS (每秒输入事件数)
    events_out_rate: float = 0.0      # EPS (每秒输出事件数)

    # 延迟指标 (Gauge)
    collection_lag_ms: int = 0        # 采集延迟 (毫秒)
    last_event_timestamp: float = 0  # 最新事件时间戳

    # 健康指标 (Gauge)
    dlq_size: int = 0                 # 当前 DLQ 大小
    datasource_health: float = 1.0    # 数据源健康状态 (0-1)

    # 时间戳
    last_updated: float = 0          # 最后更新时间
```

### Task 2: 创建 Metrics Collector
**文件:** `src/collection/metrics/collector.py`

**功能:**
- 收集各类采集指标
- 计算 EPS (使用滑动窗口)
- 更新健康状态
- 线程安全的指标更新

### Task 4: 集成到 Vector 配置
**文件:** `collector/configs/vector.toml` (修改)

**新增 sources:**
```toml
[sources.internal_metrics]
type = "internal_metrics"
```

**新增 sinks:**
```toml
[sinks.prometheus_metrics]
type = "prometheus_exporter"
inputs = ["internal_metrics", "collection_metrics"]
default_namespace = "secalert"
```

### Task 5: 创建 Metrics API
**文件:** `src/api/metrics_endpoints.py` (新增部分)

**Endpoints:**
```
GET /api/v1/metrics              # Prometheus 格式

```
