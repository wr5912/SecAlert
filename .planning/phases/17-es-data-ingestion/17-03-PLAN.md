**修改后的 logstash 服务定义:**
```yaml
logstash:
    image: docker.elastic.co/logstash/logstash:8.11.0
    container_name: secalert-logstash
    volumes:
      - ./logstash/pipeline:/usr/share/logstash/pipeline:ro
      - ./logstash/config/logstash.yml:/usr/share/logstash/config/logstash.yml:ro
    ports:
      - "5044:5044"
    environment:
      - "LS_JAVA_OPTS=-Xms2g -Xmx2g"
      - "LOGSTASH_ENABLED=${LOGSTASH_ENABLED:-false}"
    networks:
      - secalert-network
    depends_on:
      kafka:
        condition: service_healthy
    healthcheck:
      test: ["CMD-SHELL", "curl -s http://localhost:9600/_node/stats || exit 1"]
      interval: 30s
      timeout: 10s
      retries: 5
      start_period: 60s
```

说明：
- 添加 healthcheck 使用 Logstash API (`http://localhost:9600/_node/stats`)
- `start_period: 60s` 给予 Logstash 足够启动时间
- `LOGSTASH_ENABLED` 环境变量控制是否启动
- depends_on 使用 condition: service_healthy 确保 Kafka 先就绪
</action>

   docker-compose up -d logstash
