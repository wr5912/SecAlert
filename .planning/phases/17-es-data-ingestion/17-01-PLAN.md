<objective>
**Purpose:** 在 Docker Compose 中添加 Logstash 服务，配置从 Elasticsearch 拉取数据的 pipeline

**Key Decisions from Logstash 调研:**
- 使用 Logstash 而非 Python elasticsearch 库（Vector 不支持 ES 作为 source）
- Logstash elasticsearch input 插件支持 Scroll API 深度分页
- Docker 容器化部署
- 幂等写入（docinfo + document_id）防止重复数据
- JVM 内存锁死避免 OOM
</objective>

  <action>
在 docker-compose.yml 中添加 Logstash 服务：

```yaml
  logstash:
    image: docker.elastic.co/logstash/logstash:8.11.0
    container_name: secalert-logstash
    volumes:
      - ./logstash/pipeline:/usr/share/logstash/pipeline:ro
      - ./logstash/config/logstash.yml:/usr/share/logstash/config/logstash.yml:ro
    ports:
      - "5044:5044"
    environment:
      - "LS_JAVA_OPTS=-Xms2g -Xmx2g"
    networks:
      - secalert
    depends_on:
      - kafka
```

注意：
- JVM 内存锁死 `-Xms2g -Xmx2g`（避免 OOM）
- 使用与 ES 8.x 兼容的 Logstash 8.11.0
- 从 `./logstash/pipeline` 挂载 pipeline 配置

<task type="auto">
  <name>Task 2: 创建 Logstash 配置目录结构</name>

创建目录和配置文件：

1. `logstash/config/logstash.yml`:
```yaml
http.host: "0.0.0.0"
xpack.monitoring.enabled: false
api.enabled: false
```

2. `logstash/pipeline/es-input.conf` (初始版本):
```ruby
input {
  # ES input 插件 - 用于从已有 ES 拉取数据
  elasticsearch {
    hosts => ["${ES_HOST:localhost:9200}"]
    user => "${ES_USER:elastic}"
    password => "${ES_PASSWORD:changeme}"
    index => "${ES_INDEX:security-alerts-*}"
    scroll => "5m"
    size => 2000
    docinfo => true
    query => '{"query": { "match_all": {} }}'
  }
}

filter {
  # 标准化时间戳
  date {
    match => ["@timestamp", "ISO8601"]
    timezone => "Asia/Shanghai"
    target => "@timestamp"
  }

  # 移除 ES 6.x 的 _type metadata（ES 8.x 不兼容）
  mutate {
    remove_field => ["[@metadata][_type]"]
  }
}

output {
  # 输出到 Kafka（接入系统）
  kafka {
    bootstrap_servers => "${KAFKA_BOOTSTRAP:kafka:9092}"
    topic_id => "raw-events"
    codec => json
    document_id => "%{[@metadata][_id]}"
  }

  # 调试用（完成后注释掉）
  stdout { codec => rubydebug }
}
```

注意：
- 使用 Scroll API (`scroll => "5m"`) 深度分页
- `docinfo => true` 保留原始 _id
- `document_id => "%{[@metadata][_id]}"` 实现幂等写入
- 输出到 Kafka topic `raw-events`
</action>
