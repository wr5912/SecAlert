    修复 templateId 异步传递的 race condition：使用 await 确保模板保存完成后再更新 UI

    1. 修改 AIDetectPanel.tsx 的 mutation onSuccess 回调：
       - 使用 async/await 确保模板保存完成
       - 模板保存成功后才能触发后续的 MappingPreview 更新

       ```typescript
       onSuccess: async (data) => {
         // 1. 先更新识别结果（立即显示）
         setAiRecognitionResult(data);
         setFieldMappings(data.field_mappings || {});

         // 2. 异步保存模板 - 使用 await 确保完成
         const templateName = `AI识别-${data.detected_format}-${Date.now()}`;
         try {
           const response = await fetch('/api/ingestion/templates', {
             method: 'POST',
             headers: { 'Content-Type': 'application/json' },
             body: JSON.stringify({
               name: templateName,
               device_type: 'other',
               connection: { host: '', port: 514, username: '', password: '', protocol: 'syslog' },
               log_format: data.detected_format,
               custom_regex: data.regex_pattern
             }),
           });
           if (response.ok) {
             const template = await response.json();
             // 3. 模板保存成功后更新 templateId
             setCurrentTemplateId(template.id);
             setCurrentTemplateSaved(true);
           }
         } catch (e) {
           console.error('Auto-save template failed:', e);
         }

         setErrorMessage(null);
         onSuccess?.(data);
       }
       ```
