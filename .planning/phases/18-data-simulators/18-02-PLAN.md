# Phase 18 Plan 02: DLQ 死信队列机制

## 执行目标

实现 SM-01 死信队列 (DLQ) 机制：解析失败日志自动路由到 DLQ Topic，支持重试、归档和重处理。

## 需求回顾 (SM-01)

| 需求 | 说明 |
|------|------|
| DLQ Topic | 解析失败日志自动路由到 DLQ Topic |
| 重试机制 | 默认 3 次，间隔 1min/5min/30min |
| MinIO 归档 | 超过重试上限后归档到 MinIO |
| 告警通知 | DLQ 告警通知（管理员通知） |
| 重处理 | DLQ 消息重处理（人工/自动） |

### Task 2: 创建 DLQ Consumer 服务
**文件:** `src/collection/dlq_consumer.py`

**功能:**
- 消费 `dlq-events` topic
- 解析失败消息并记录原始内容和失败原因
- 发送到重试队列或最终归档

**重试策略:**
- Retry 1: 延迟 1 min
- Retry 2: 延迟 5 min
- Retry 3: 延迟 30 min
- Retry 4+: 归档到 MinIO

**消息格式 (DLQ):**
```json
{
  "original_event": { ... },
  "error": "ParseError: Invalid JSON",
  "source": "vector-syslog",
  "retry_count": 0,
  "first_failure": "2026-04-03T12:00:00Z",
  "last_retry": null,
  "status": "pending"
}
```

### Task 3: 创建 DLQ Redis 状态管理
**文件:** `src/collection/dlq_state.py`

**功能:**
- 使用 Redis 存储 DLQ 消息状态
- Key: `dlq:{message_id}`
- TTL: 7 天

**字段:**
- `retry_count`: 重试次数
- `next_retry`: 下次重试时间
- `status`: pending / retrying / archived / reprocessed
- `original_topic`: 原始 topic
- `partition`: 分区号
- `offset`: 原始偏移量

### Task 4: 实现延迟重试机制
**文件:** `src/collection/dlq_retry.py`

**实现:**
- 使用 Redis Sorted Set 实现延迟队列
- Score = 重试时间戳
- 定时扫描可重试消息

**处理流程:**
```
1. ZRANGEBYSCORE dlq:retry_queue 0 NOW
2. 对每条消息:
   a. 调用解析器重试
   b. 成功 → 确认 → 更新状态为 reprocessed
   c. 失败 → retry_count++
   d. retry_count < 3 → ZADD 下次重试时间
   e. retry_count >= 3 → 归档到 MinIO
```

### Task 5: MinIO 归档
**文件:** `src/collection/dlq_archive.py`

**功能:**
- 将超过重试上限的消息归档到 MinIO
- 归档路径: `dlq-archive/{YYYY-MM-DD}/{message_id}.json`
- 归档元数据: `dlq-archive/{YYYY-MM-DD}/{message_id}.meta.json`

**归档内容:**
```json
{
  "message_id": "uuid",
  "original_event": { ... },
  "error": "ParseError",
  "retry_count": 3,
  "first_failure": "2026-04-03T12:00:00Z",
  "archived_at": "2026-04-03T12:31:00Z",
  "archive_path": "dlq-archive/2026-04-03/uuid.json"
}
```

### Task 6: 告警通知
**文件:** `src/collection/dlq_notifier.py`

**触发条件:**
- DLQ 消息数量超过阈值 (默认 100)
- DLQ 消息堆积超过时间 (默认 1 hour)
- 归档操作完成

**通知方式:**
- 日志警告 (INFO level)
- 未来扩展: 邮件/Webhook 通知

### Task 7: DLQ 重处理 API
**文件:** `src/api/routes.py` (新增)

**Endpoints:**
```
GET  /api/v1/dlq                    # 列出 DLQ 消息
GET  /api/v1/dlq/{id}               # 查看详情

POST /api/v1/dlq/reprocess-all      # 重处理所有 pending

```
