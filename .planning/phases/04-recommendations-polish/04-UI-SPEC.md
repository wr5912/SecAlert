## Component Inventory

### AlertList (告警列表组件)

**Purpose:** 显示 Critical/High 告警摘要列表，支持筛选

**States:**
- Default: 显示告警列表，每条包含严重度标签、攻击源IP、目标资产、主要行为摘要
- Empty: 显示 empty state (暂无需要处理的告警)
- Error: 显示 error state (加载失败，请稍后重试)
- Loading: 骨架屏占位

**Interactions:**
- 点击告警行 → 打开 AlertDetail 单屏
- 支持 severity 筛选下拉 (Critical, High, Medium, Low, All)

### AlertDetail (告警详情单屏组件)

**Purpose:** 单屏展示告警完整信息（攻击链摘要 + 处置建议 + 操作按钮）

**Layout (从上到下):**
1. 严重度标签 + 攻击链ID
2. ChainTimeline (简化线性时间线)
3. RemediationPanel (处置建议面板)
4. 操作按钮区 (Primary CTA + Secondary CTA + 备注字段)

**States:**
- Default: 完整展示
- Loading: 骨架屏
- Error: 错误提示 + 重试按钮

**Interactions:**
- "确认已通报" → 记录响应 + 刷新列表
- "确认为误报" → 显示确认弹窗 → 恢复告警至活跃状态
- 可展开"技术详情"查看 ATT&CK 引用

### ChainTimeline (简化攻击链时间线组件)

**Purpose:** 以线性方式展示简化后的攻击链关键节点

**Layout:** 水平排列的 4 个节点
1. 攻击源 IP (图标: 放大镜)
2. 主要攻击行为 (图标: 警告)
3. 受影响资产 (图标: 靶心)
4. 攻击阶段 (图标: 定位)

**States:**
- Default: 显示 4 个节点 + 摘要文字
- Loading: 骨架屏

### RemediationPanel (处置建议面板组件)

**Purpose:** 展示处置建议，包含一行核心行动 + 可展开详细说明

**Layout:**
- 主行: 核心行动描述 (加粗，引用具体资产信息)
- 可展开区: 详细步骤列表 + ATT&CK 引用

**States:**
- Default: 主行 + "查看详情" 折叠按钮
- Expanded: 展开详细步骤
- Template: 来源为模板的标记
- LLM: 来源为 LLM 生成的标记

**Interactions:**
- 点击"查看详情" → 展开/收起详细步骤

## Interaction Patterns

### 告警响应流程

```
1. 用户在 AlertList 点击告警
   → AlertDetail 单屏打开

2. 用户阅读处置建议
   → 可点击"查看详情"展开 ATT&CK 技术引用

3. 用户选择操作:
   a) "确认已通报" (Primary CTA, accent 色)
      → 显示成功提示
      → 告警标记为已响应
      → 自动返回列表

   b) "确认为误报" (Destructive, red 色)
      → 显示确认弹窗: "确认为误报：此操作将恢复告警至活跃状态"
      → 用户确认 → 调用 restore_chain API → 返回列表
      → 用户取消 → 关闭弹窗

4. 用户可添加可选备注
```

### 筛选交互

- Default: 只显示 Critical + High (D-06)
- 点击筛选下拉 → 选择 severity 级别
- 选择 "All" → 显示全部告警

### 误报恢复入口

- AlertList 提供"已抑制告警" Tab/入口
- 用户可查看被 Phase 3 错误抑制的告警
- 支持恢复操作 (确认为误报)
